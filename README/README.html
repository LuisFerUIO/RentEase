NOTAS DEL PROYECTO

  • La estrella del proyecto es JavaScript. Aunque el diseño sea atractivo, si no hay lógica funcional, la calificación será cero.

  • Se considerarán como puntos extra las siguientes funcionalidades:

    • Solicitud de nueva sesión luego de 60 minutos de inactividad.

    • Eliminación de toda la información del usuario al resetear la contraseña.

  • Cada usuario podrá editar, guardar, eliminar y marcar como favorito únicamente sus propios "flats"; no podrá hacerlo con los de otros usuarios.

  • La página de inicio será la de login.

  • El diseño no requiere ser muy elaborado.

  • Se evaluarán las validaciones implementadas.

  • La selección de fecha debe contar con un calendario.

  • La actualización de la cuenta se realizará mediante un modal.

  • No es necesario incluir imágenes.

  • Deben usarse nombres claros y coherentes para las variables.

  • Se utilizará una variable (por ejemplo, loget) para determinar si el usuario está logueado. Esta variable permitirá controlar el acceso a las funciones de modificación.

  • Si el usuario intenta acceder sin estar logueado (es decir, sin que exista la variable loget), será redirigido automáticamente al login.
